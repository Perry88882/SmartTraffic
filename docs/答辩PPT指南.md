### 讲解提示词

> "首先看项目背景。网络流量分析面临三大痛点。"
> "第一，Wireshark 需要专家逐个包手动看。第二，HTTPS 加密占比已超 95%，传统 DPI 失效。第三，市面上缺少把实时分析、安全评估、历史回溯、可视化集成到一起的平台。"
> "右边这张表对比了四种主流方法。端口分类准确率不到 60%。DPI 虽然在明文流量上准确率很高，但碰到加密流量就完全失效。深度学习准确率最高但实时性差。所以我们的方案是统计特征加机器学习——在准确率和实时性之间取最佳平衡。"

### 设计说明

- **左半页：** 三个痛点用红色叉号（`#ff4757`）图标强调
- **右半页：** 表格，推荐行（统计+ML）用青色高亮
- **底部结论：** 用色块背景标注

## 第 6 页：数据管道 — 核心流程（1分钟）

### 布局

```
┌─────────────────────────────────────────────────────────────┐
│              数据管道：六阶段流水线                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ①抓包      ②解析      ③流重组      ④特征       ⑤AI推理    ⑥安全
│  ┌──┐     ┌──┐      ┌──┐      ┌──┐      ┌──┐      ┌──┐   │
│  │  │ ──→ │  │ ──→  │  │ ──→  │  │ ──→  │  │ ──→  │  │   │
│  └──┘     └──┘      └──┘      └──┘      └──┘      └──┘   │
│   │        │         │         │          │         │      │
│   ▼        ▼         ▼         ▼          ▼         ▼      │
│ scapy   提取18    五元组    30维向量   RF模型     IP情报    │
│ sniff   个字段    哈希表    归一化     启发式     8维风险   │
│ Npcap   MAC地址   超时清理   log缩放   随机兜底   GeoIP     │
│         TTL       TCP/60s                      定位       │
│         flags     UDP/30s                                 │
│                                                             │
│  特点：模块独立 │ 队列解耦 │ 可替换 │ 可扩展                   │
└─────────────────────────────────────────────────────────────┘
```

### 讲解提示词

> "数据管道是系统最核心的设计，它把流量分析拆成六个独立的处理阶段。"
> "第一阶段——scapy 加 Npcap 从网卡抓包。第二阶段——解析器提取 MAC、IP、TTL、TCP 标志位等 18 个字段。第三阶段——按五元组把离散数据包聚合成网络流，TCP 流 60 秒超时自动清理。第四阶段——从流中提取 30 维统计特征，包含包长分布、时间间隔、端口编码等。第五阶段——三级分类器输出应用标签和置信度。第六阶段——安全引擎评估风险并查询 IP 地理位置。"
> "管道最大的好处是模块独立——想换分类模型？只改第五阶段。想加 DPI 检测？在第二阶段后插入。每个模块有明确的输入输出契约。"

### 设计说明

- **六个方框：** 横向排列，箭头连接
- **每个方框：** 上方编号+名称，下方标注输入/关键技术
- **底部：** 一行总结特点

> "五层解析的结果就像这样——一个完整的数据包解剖图。从链路层的 MAC 地址和帧类型，到网络层的 IP、TTL、分片标志，到传输层的端口和 TCP 标志位，再到应用层的载荷大小和 AI 分类。每一层的关键字段都被提取出来，存储到数据库，可以在前端逐层展开查看。解析器只有一百三十行代码，但兼容了 scapy 2.4 到 2.6 的所有版本。"

## 第 8 页：AI 分类器 — 三级降级（1分钟）

### 布局

```
┌─────────────────────────────────────────────────────────────┐
│           AI 分类器：三级弹性降级策略                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                     ┌──────────┐                            │
│                     │ 30维特征  │                            │
│                     │ 输入向量  │                            │
│                     └────┬─────┘                            │
│                          │                                  │
│               ┌──────────▼──────────┐                       │
│               │  Level 1            │                       │
│               │  RandomForest 模型   │  ← 100棵树             │
│               │  predict_proba()    │    max_depth=12       │
│               │  准确率 ~85%         │    有模型时使用        │
│               └────┬─────────┬──────┘                       │
│                    │         │                              │
│                 成功│         │失败/无模型                    │
│                    │    ┌────▼──────────┐                   │
│                    │    │  Level 2       │                   │
│                    │    │  启发式规则     │  ← 7条规则         │
│                    │    │  准确率 ~75%    │    基于协议行为     │
│                    │    └────┬─────┬─────┘                   │
│                    │         │     │                         │
│                    │      成功│     │失败                     │
│                    │         │  ┌──▼─────────┐              │
│                    │         │  │  Level 3    │              │
│                    │         │  │  随机兜底    │  ← 永不出错   │
│                    │         │  │  准确率 ~15% │              │
│                    │         │  └────┬────────┘              │
│                    ▼         ▼       ▼                       │
│               ┌──────────────────────────┐                   │
│               │  { label, confidence }   │                   │
│               └──────────────────────────┘                   │
│                                                             │
│   启发式规则示例：                                            │
│   • DNS (53/UDP) → 网页 82%    • HTTPS+大包+高速 → 视频 85%  │
│   • 小包>60%+短间隔 → 游戏 76% • HTTP(80/TCP) → 网页 78%     │
└─────────────────────────────────────────────────────────────┘
```

### 讲解提示词

> "分类器最大的设计特色是三级弹性降级。"
> "第一级，尝试加载 RandomForest 模型——100 棵树，最大深度 12——做 predict_proba 推理。训练数据量较小的时候，深度限制能有效防止过拟合。"
> "如果模型文件不存在或加载失败，自动降级到第二级——七条启发式规则。比如 DNS 端口 53 的 UDP 查询大概率是网页浏览的前置步骤，HTTPS 加上高速率和大包比例判定为视频流，高频小包加短间隔判定为在线游戏。这些规则基于真实的网络协议行为设计，准确率大约百分之七十五。"
> "如果连规则也匹配不了，最后用随机分类兜底——虽然准确率低，但保证了系统在任何情况下都不会崩溃。"

### 设计说明

- **流程图：** 从上到下，分支清晰
- **三级用不同颜色：** Level 1 青色、Level 2 紫色、Level 3 灰色
- **底部规则示例：** 4 个小色块，每个一条规则
